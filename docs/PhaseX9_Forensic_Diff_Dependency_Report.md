## 8. Why Legacy Round-Trips Perfectly

The legacy workbook round-trips because **ConMas generates the workbook in the same format it expects to read back**:

1. **Generate**: ConMas creates Sheet1 with 5 merged regions, cell values, **PrintArea**, and **6 comments** with full metadata (name, type, cluster index, parameters).
2. **Read back**: ConMas opens the workbook, reads **visible sheets only**, processes Sheet1's **PrintArea** range, reads **comments** from the content cells, reconstructs 6 clusters from comment metadata.

## 9. Why Our Workbook Produces Only 1 Cluster (Root Cause Chain)

```
1. WorkbookGenerator sets no PrintArea on Sheet1
     ↓
2. WorkbookGenerator writes comments to full merge ranges (e.g., "$A$1:$B$2")
     → Excel COM throws COMException on AddComment for multi-cell ranges
     → 5 of 6 comments silently fail
     → Only A12 (single cell) gets a comment
     ↓
3. WorkbookGenerator creates no merged cells on Sheet1
     → Sheet1 is empty (0 cells, 0 merges)
     ↓
4. WorkbookReaderService opens the generated workbook
     → Finds _Fields sheet (hidden) with 6 data rows
     → COM UsedRange on hidden sheet may return incorrect row count
     → OR: _Fields is read but some other issue in BuildClusterFromDefinition
     → Result: only 1 cluster reconstructed
```
