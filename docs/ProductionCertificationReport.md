## 1. Validation Pipeline

```
Original Excel (.xlsx)
        │
        ▼
  ┌─────────────────┐
  │  COM Capture     │  ExcelCaptureService → WorkbookDefinition
  └────────┬────────┘
           ▼
  ┌─────────────────┐
  │  WorkbookDef     │  Canonical domain model (source of truth)
  └────────┬────────┘
           ▼
  ┌─────────────────┐
  │  Browser Render  │  RuntimeForm → Frontend Editor
  └────────┬────────┘
           ▼
  ┌─────────────────┐
  │  Save Edited     │  POST /api/form/save-edited
  └────────┬────────┘
           ▼
  ┌─────────────────┐
  │  WorkbookValue   │  Surgery: writes ONLY editable cell values
  │  Writer          │  Preserves everything else
  └────────┬────────┘
           ▼
  ┌─────────────────┐
  │  WorkbookDiff    │  30+ category validator
  │  Validator       │  Fails on ANY structural change
  └────────┬────────┘
           ▼
  ┌─────────────────┐
  │  Download .xlsx  │  Only if validation PASSES
  └─────────────────┘
```

## 8. Known Limitations

| # | Limitation | Impact | Target Resolution |
|---|------------|--------|-------------------|
| 1 | WorkbookValueWriter only writes cell values — does not insert/delete rows | Cannot add/remove fields after save | Phase 6 |
| 2 | Conditional formatting rules compared at part level — not rule-by-rule | Part hash diff if any rule changes | Phase 6 |
| 3 | VBAProjectPart preserved but binary hash must match | Any VBA modification fails validation | Phase 6 |
| 4 | PivotTable parts checked by relationship count only | Structural count check, not content | Phase 6 |
| 5 | External link relationships preserved as-is | Cannot update external link targets | Phase 6 |
