## Differences Found

### Difference 1 — Comment Format (FIXED)

| Aspect | Legacy | Before Fix | After Fix |
|--------|--------|------------|-----------|
| Line 0 | Field Name | `{cluster.Name}` | `{cluster.Name}` ✅ |
| Line 1 | Field Type | `{cluster.Type}` | `{cluster.Type}` ✅ |
| Line 2 | (empty) | `{cluster.ClusterId}` | `(empty)` ✅ |
| Line 3 | Parameters / "0" | `{paramStr}` | `{paramStr}` ✅ |
| Default empty params | `"0"` | `""` | `"0"` ✅ |

**Root cause:** The original implementation used `ClusterId` on line 2 (as a logical "index") instead of an empty line matching the legacy format.

**Fix applied in:** `WorkbookGenerator.cs`, `WriteCellComments()` method.

### Difference 2 — _Fields Sheet Order (MINOR)

Legacy workbooks place `_Fields` as the **first** sheet (sheet1.xml) and the content sheet as second (sheet2.xml). Our implementation adds `_Fields` as the **last** sheet (`After: workbook.Worksheets[last]`). 

- **Impact:** None. Sheet order is cosmetic. Both are functionally identical.
- **Fix:** Not needed unless republish compatibility depends on sheet order. If needed, change `After:` to `Before: workbook.Worksheets[1]`.

### Difference 3 — Cell Borders (KNOWN GAP)

The `CellStyleInfo` model has border properties (`BorderTop`, `BorderBottom`, `BorderLeft`, `BorderRight`) but `ApplyStyleToRange()` does not apply them.

- **Impact:** Generated workbook lacks cell borders that may exist in the original template.
- **Status:** Pre-existing gap, not introduced by this implementation.

## Comment Format Validation

### Legacy Comment (at cell A1)

Decoded from OOXML:
```
samples\r\nKeyboardText\r\n\r\n0\r\n0\r\n(empty trailing lines...)
```

As a single string:
```
samples
KeyboardText

0
0
```

Line breakdown:
```
[0] = "samples"           ← Field name
[1] = "KeyboardText"       ← Field type
[2] = ""                   ← (empty line)
[3] = "0"                  ← First parameter
[4] = "0"                  ← Second parameter
[5+] = ""                  ← Trailing empty lines
```

### New Comment (after fix)

```
{cluster.Name}\n{cluster.Type}\n\n{paramStr}
```

Line breakdown:
```
[0] = cluster.Name         ← Field name
[1] = cluster.Type         ← Field type
[2] = ""                   ← (empty line — matches legacy)
[3] = "0" or paramStr      ← Parameters (matches legacy)
```

**Trailing empty lines:** Not included. The ConMas comment parser reads line by line and would stop at non-empty lines. The trailing empty lines in the legacy are artifacts of the original designer and not functionally significant.

## Fix Applied

### File: `ExcelAPI/ExcelAPI/Generators/WorkbookGenerator.cs`

**Change:** Comment format adjusted to match legacy ConMas format.

```csharp
// BEFORE (Phase X.1 implementation):
string commentText = $"{cluster.Name}\n{cluster.Type}\n{cluster.ClusterId}\n{paramStr}";

// AFTER (Phase X.2 fix):
string paramStr = "0";  // Changed default from "" to "0"
// ... build paramStr from InputParameters if non-empty ...
string commentText = $"{cluster.Name}\n{cluster.Type}\n\n{paramStr}";
```

**What changed:**
1. Line 2: `{cluster.ClusterId}` → empty line (matches legacy format)
2. Default for `paramStr`: `""` → `"0"` (matches legacy placeholder)

**Build verification:** Zero C# compilation errors.
