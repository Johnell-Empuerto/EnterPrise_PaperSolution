## 2. Root Cause

### The bug is in `FormController.cs`, line 146

```csharp
// FormController.cs — FromExcel()
string previewUrl = $"/preview/page_{templateId}.png";
```

This creates a URL that **guesses** the filename format using the **upload GUID** (`templateId`). But `CapturePrintAreaAsync()` internally generates a **completely different random GUID** for the PNG filename.

The actual PNG is saved at:
```
Preview/page_{captureGuid}.png
```
where `captureGuid` is generated inside `CapturePrintAreaAsync()` at line 507:
```csharp
string fileId = Guid.NewGuid().ToString("N");
string previewFileName = $"page_{fileId}.png";
```

The `FromExcel()` method has no access to `fileId` — it's a local variable inside `CapturePrintAreaAsync()`.

### Why previewUrl is Wrong

The capture result DOES contain the correct path via `captureResult.ImageUrl`:
```csharp
return new CaptureResult
{
    ImageUrl = $"/preview/{previewFileName}",  // CORRECT: /preview/page_{captureGuid}.png
    ...
};
```

But `FromExcel()` on line 146 ignores `captureResult.ImageUrl` and constructs its own URL:
```csharp
string previewUrl = $"/preview/page_{templateId}.png";  // WRONG: uses upload GUID
```

### Why backgroundImage is Also Wrong (Minor)

`ConvertCaptureToForm()` copies the correct preview to a new location:
```csharp
string bgFileName = $"bg_{Guid.NewGuid():N}.png";  // YET ANOTHER GUID
```

This creates a third GUID for the same image. The `Forms/bg_{}.png` copy is functional (it serves as a "permanent" copy), but it introduces unnecessary filename divergence.

## 5. Frontend URL Construction

From `FormPage.tsx`:

```tsx
const bgUrl = `${API_BASE_URL}/preview/page_${templateId}.png`;
```

The frontend **reconstructs** the URL from `templateId` instead of consuming the backend-provided `previewUrl`. Even if the backend returned the correct URL, the frontend wouldn't use it because it's hardcoded to use templateId.

---

## 6. The Fix

### Fix 1: Backend — Use the Actual Generated Preview Path

In `FormController.cs`, change `FromExcel()` to use the actual image path from the capture result instead of constructing a URL from templateId:

**Line 146:**
```csharp
// BEFORE (broken):
string previewUrl = $"/preview/page_{templateId}.png";

// AFTER (correct):
string previewUrl = captureResult.ImageUrl;  // Already contains "/preview/page_{fileId}.png"
```

This single change makes `previewUrl` point to the actual file.

### Fix 2: Frontend — Use Backend-Provided previewUrl

In `FormPage.tsx`, change to consume the backend-provided URL instead of reconstructing from templateId. This requires passing `previewUrl` through from the upload result to the FormPage component.

However, since the frontend currently stores the previewUrl in `runtimeUploadResult` but never uses it (the FormPage reconstructs from templateId), we could either:
- Pass `previewUrl` as a prop to FormPage
- Or fix the backend to use the same GUID for both templateId and PNG filename

### Fix 3: Architectural (Single Source of Truth)

The cleanest fix: make `CapturePrintAreaAsync()` use the **templateId** (or accept it as a parameter) for the PNG filename instead of generating a new GUID. This ensures `page_{templateId}.png` actually exists:

```csharp
// In ExcelCaptureService.CapturePrintAreaAsync():
// Accept templateId as a parameter and use it for the filename
string previewFileName = $"page_{templateId}.png";
```

This would make `previewUrl = /preview/page_{templateId}.png` correct, and `FormPage.tsx`'s hardcoded URL would also work without modification.

## 8. Recommended Fix

**Minimal change**: In `FormController.cs` line 146, replace the constructed `previewUrl` with the actual image path from the capture result:

```csharp
string previewUrl = captureResult.ImageUrl;  // "/preview/page_{captureGuid}.png"
```

Then pass `previewUrl` to the frontend via the `runtimeUploadResult` and use it in `FormPage.tsx` instead of reconstructing from `templateId`.

**Architectural fix (recommended)**: Modify `CapturePrintAreaAsync()` to accept the `templateId` as an optional parameter and use it for the PNG filename. This ensures the same GUID identifies both the workbook and the preview:

```csharp
// Accept templateId for consistent filenaming
public async Task<CaptureResult> CapturePrintAreaAsync(
    string excelFilePath, 
    string? fileId = null,  // New parameter
    CancellationToken cancellationToken = default)
{
    string id = fileId ?? Guid.NewGuid().ToString("N");
    string previewFileName = $"page_{id}.png";
    ...
}
```

This way, `FromExcel()` passes the templateId as the fileId, and `previewUrl = /preview/page_{templateId}.png` becomes correct automatically.
